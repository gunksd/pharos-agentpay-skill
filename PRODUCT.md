# Product

## Register

product

## Users

- **Hackathon judges** (primary, first 72h): skim the live dApp to verify the skill works on-chain; need zero-friction read access, instant credibility signals (real txs, verified contract).
- **Agent operators / developers**: connect a wallet on Pharos Atlantic to post bounty-backed tasks, claim work, settle payments; check counterparty trust scores before doing business.

## Product Purpose

Web frontend for the AgentPay Escrow skill: a live job board for the agent labor market on Pharos. Reads the deployed escrow contract directly (task lifecycle, reputation ledger) and lets a connected wallet drive every contract action (post / claim / submit / approve / reject / force-settle / cancel). Success: a judge can browse real on-chain tasks in <5s without a wallet, and a developer can run a full escrow cycle from the page.

## Brand Personality

Lighthouse keeper's night watch: calm, dependable, quietly playful. The ink-drawn cat mascot (flying with a lantern) carries the charm; the interface around it stays precise and trustworthy. Three words: **steadfast, luminous, mischievous**.

## Anti-references

- Generic DeFi dashboard: navy + neon-green candles, dense stat grids, hero metrics.
- SaaS landing slop: cream background, purple gradient text, identical icon-card grids, eyebrow labels over every section.
- Kawaii overload: the mascot is the single playful element; the UI itself never uses rounded-bubble typography or pastel rainbows.

## Design Principles

1. **The lantern is the light**: one warm amber accent carries every primary action and "open" state; everything else stays in the night palette.
2. **Proof over promise**: real contract data, real tx hashes, verified-source links; never mock numbers.
3. **Readable in the dark**: dark theme tuned for contrast (AA minimum), data in mono, prose capped at 70ch.
4. **The tool disappears into the task**: standard affordances for forms, tables, wallets; charm lives in the mascot and copy, never in widget shapes.
5. **One screen, whole story**: board, post form, and reputation lookup reachable without routing depth.

## Accessibility & Inclusion

WCAG AA: 4.5:1 body text, 3:1 large text; visible focus rings; reduced-motion alternatives for all animation; status conveyed by label + color, never color alone.
